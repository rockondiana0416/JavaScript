What is it?

React is a light that turns on your javaScript to make it bigger, better, brighter, and bolder. It is an open source library that is easy to learn and use, easy to maintain, and easy to debug. React is a library of code made of components. The components are like the books at you local library. You don't have to use the whole thing like you would a framework. Being that it is a library, pieces of code called snipits can be used. Using snipits in your site will make it faster and makes for a great user experience.

React has an extension called JSX that creates a virtual DOM that can be manipulated using HTML code. By doing that, it can render code concurrently. Concurrently meaning, it can pause other code to make high priority code update first. This all makes your seach boxes, buttons, input boxes, and menus super fast. React Community

There is a ton of support for the React community. Again just head over to reactjs.org and find their Community Section. There are community forums, discussions, chats and news. Hackermoon has a large selection of articles and tutorials on how to use and implement React code. Because React code is open source, it can be shared and reshared by anyone.

Examples and Favorites

Take a look at Facebook.com. Facebook was coded using React.js. The creator of React Jordan Walke, an engineer at Facebook, wanted to improve the UI of the site. So in 2011 it was released to the public as open source and free. React creates a better user experience by using components of code. Components are the small snipits of code that can be injected into your html.

Because react is used on big name websites like Air BNB, BBC, Dropbox, Flipboard, and Netflix. Clients and employers feel confident in developers that know react.
